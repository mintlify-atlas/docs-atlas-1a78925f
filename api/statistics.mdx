## Overview

The Statistics API provides functionality to export alert data in CSV format for external analysis and reporting. This API generates a download URL for a CSV file containing all alert history.

<Note>
  Statistics operations use the `/api/panel/graphql` endpoint.
</Note>

## Query: exportAlertsCsvUrl

Generate a CSV export URL for all alerts in the system.

From `~/workspace/source/src/pages/Estadisticas.tsx:22-31`:

```graphql
query {
  exportAlertsCsvUrl
}
```

### Request

<CodeGroup>

</CodeGroup>

### Response

<ResponseField name="exportAlertsCsvUrl" type="string" required>
  URL to download the generated CSV file containing alert data
  
  The URL may be:
  - A direct download link to a pre-signed S3 URL
  - A relative path to a generated file on the server
  - An absolute URL to the CSV resource
</ResponseField>

### Response Example

### Complete Export Handler

From `~/workspace/source/src/pages/Estadisticas.tsx:13-61`:

```typescript
const [isExporting, setIsExporting] = useState(false);

const handleExportCSV = async () => {
  setIsExporting(true);
  try {
    toast({
      title: "Preparando exportación",
      description: "Generando archivo CSV...",
    });

    // Get the download URL from GraphQL
    const query = `
      query {
        exportAlertsCsvUrl
      }
    `;

    const result = await fetchGraphQL<{ exportAlertsCsvUrl: string }>(
      "/panel/graphql",
      query
    );

    const downloadUrl = result.exportAlertsCsvUrl;

    console.log("URL de descarga:", downloadUrl);

    // Download the file
    const link = document.createElement("a");
    link.href = downloadUrl;
    link.download = `alertas_${new Date().toISOString().split('T')[0]}.csv`;
    link.target = "_blank";
    document.body.appendChild(link);
    link.click();
    document.body.removeChild(link);

    toast({
      title: "Exportación exitosa",
      description: "El archivo CSV ha sido descargado correctamente",
    });
    
  } catch (error) {
    console.error("Error exportando CSV:", error);
    toast({
      title: "Error",
      description: "No se pudo exportar el archivo CSV",
      variant: "destructive",
    });
  } finally {
    setIsExporting(false);
  }
};
```

## CSV File Format

Based on the alert data structure, the exported CSV file contains the following columns:

<ResponseField name="id" type="string">
  Unique identifier for the alert
</ResponseField>

<ResponseField name="alertType" type="string">
  Type of alert (e.g., "Exceso de velocidad")
</ResponseField>

<ResponseField name="responsible" type="string">
  Area or person responsible for the alert
</ResponseField>

<ResponseField name="priority" type="string">
  Priority level (Alta, Media, Baja)
</ResponseField>

<ResponseField name="driver" type="string">
  Name of the driver
</ResponseField>

<ResponseField name="generatingUnit" type="string">
  Vehicle or unit identifier
</ResponseField>

<ResponseField name="state" type="string">
  Current state of the alert (En espera, En proceso, Resuelto)
</ResponseField>

<ResponseField name="generationDate" type="string">
  ISO 8601 timestamp when the alert was generated
</ResponseField>

## Performance Considerations

### Large Datasets

For systems with large numbers of alerts:

1. **Server-side generation**: The CSV is generated on the server to avoid client memory issues
2. **Streaming**: Consider implementing streaming for very large exports
3. **Background jobs**: For extremely large datasets, consider asynchronous generation with email notification

### Browser Compatibility

The download implementation uses the `download` attribute and programmatic link clicking, which is supported in:

- Chrome 14+
- Firefox 20+
- Safari 10.1+
- Edge (all versions)

<Note>
  The `target="_blank"` attribute ensures the download doesn't navigate away from the current page.
</Note>
