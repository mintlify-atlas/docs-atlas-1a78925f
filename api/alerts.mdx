## Overview

The Alerts API provides access to the alert history panel, allowing you to retrieve all generated alerts with details about their type, responsible party, priority, and current state.

<Note>
  All alerts operations use the `/api/panel/graphql` endpoint.
</Note>

## Query: getAllAlerts

Retrieve all alerts from the system history.

## Client-Side Filtering

The alerts panel supports filtering by multiple criteria:

- **Type**: Filter by alert type (velocity, braking, geofence, etc.)
- **Responsible**: Filter by responsible area or person
- **Priority**: Filter by Alta, Media, or Baja
- **Unit**: Filter by specific vehicle/unit
- **Driver**: Filter by driver name
- **State**: Filter by En espera, En proceso, or Resuelto
- **Search**: Text search across all fields

<Note>
  Filtering is currently implemented client-side. The getAllAlerts query returns all alerts.
</Note>

## Date Formatting

The `generationDate` field returns ISO 8601 timestamps. Format them for display:

From `~/workspace/source/src/pages/PanelDeAlertas.tsx:114-128`:

```typescript
const formatDate = (isoDate: string) => {
  const date = new Date(isoDate);
  const day = date.getDate().toString().padStart(2, '0');
  const month = (date.getMonth() + 1).toString().padStart(2, '0');
  const year = date.getFullYear();
  const hours = date.getHours().toString().padStart(2, '0');
  const minutes = date.getMinutes().toString().padStart(2, '0');
  
  return `${day}/${month}/${year} ${hours}:${minutes}`;
};
```

Example output: `15/03/2024 14:30`

const getPriorityColor = (prioridad: string) => {

};

  });

## Pagination

<Warning>
  The current implementation does not support pagination. All alerts are returned in a single response.
</Warning>

For large datasets, consider implementing:
- Server-side pagination with `limit` and `offset` parameters
- Cursor-based pagination
- Virtual scrolling on the client side
