## Introduction

The FleetGuard360 GraphQL API provides a unified interface for querying and manipulating fleet monitoring data, alerts, and statistics. The API uses GraphQL over HTTP with JWT-based authentication.

## Base Endpoint

All GraphQL requests are sent to:

```
POST /api
```

The API is configured with a proxied endpoint structure where specific operations are routed through:
- `/api/alerts/graphql` - Alert type configuration and priority levels
- `/api/panel/graphql` - Alert history and statistics
- `/api/auth/login` - Authentication (REST endpoint)

## Authentication

The API requires JWT Bearer token authentication for all GraphQL operations. Obtain a token via the `/api/auth/login` endpoint.

<Note>
  All GraphQL requests must include an `Authorization` header with a valid JWT token.
</Note>

## Request Format

GraphQL requests are sent as POST requests with a JSON body:

<CodeGroup>
```json Request Structure
{
  "query": "query { ... }",
  "variables": {
    "key": "value"
  }
}
```

```javascript Fetch Example
const response = await fetch('/api/alerts/graphql', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    query: `
      query GetAllTipoAlertas {
        tipoAlertas {
          id
          nombre
        }
      }
    `
  })
});
```
</CodeGroup>

## Response Format

All GraphQL responses follow the standard GraphQL response structure:

<CodeGroup>
```json Success Response
{
  "data": {
    "fieldName": {
      // Response data
    }
  }
}
```

```json Error Response
{
  "errors": [
    {
      "message": "Error description"
    }
  ]
}
```
</CodeGroup>

<ResponseField name="data" type="object">
  Contains the requested data when the query succeeds
</ResponseField>

<ResponseField name="errors" type="array">
  Contains error objects when the query fails
  <Expandable title="Error Object">
    <ResponseField name="message" type="string">
      Human-readable error message
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Handling

The API implements several error handling mechanisms:

### HTTP Status Codes

<ResponseField name="200" type="OK">
  Request processed successfully (check `errors` field for GraphQL errors)
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing or invalid authentication token - triggers automatic logout
</ResponseField>

<ResponseField name="403" type="Forbidden">
  Token expired or insufficient permissions - triggers automatic logout
</ResponseField>

### Automatic Session Handling

The client automatically handles authentication errors:

```typescript
if (response.status === 401 || response.status === 403) {
  localStorage.removeItem("jwt_token");
  sessionStorage.removeItem("jwt_token");
  navigate("/", { state: { sessionExpired: true } });
  throw new Error("Session expired");
}
```

<Warning>
  When receiving 401 or 403 responses, the system automatically clears stored tokens and redirects to the login page.
</Warning>

### GraphQL Errors

GraphQL-level errors are returned in the `errors` array:

```typescript
if (result.errors) {
  throw new Error(result.errors[0]?.message || "GraphQL error");
}
```

## Rate Limiting

Currently, the API does not implement explicit rate limiting. However, best practices recommend:
- Implementing client-side debouncing for real-time searches
- Using pagination for large data sets
- Caching responses when appropriate

## API Versioning

The current API does not use explicit versioning. All endpoints use the latest stable schema. Breaking changes will be communicated through documentation updates.

## Common Patterns

### Using the GraphQL Hook

The application provides a custom React hook for GraphQL operations:

```typescript
import { useGraphQL } from "@/hooks/use-graphql";

const { fetchGraphQL } = useGraphQL();

const result = await fetchGraphQL<ResponseType>(
  "/alerts/graphql",
  query,
  variables
);
```

### TypeScript Types

All API responses should be typed for type safety:

```typescript
interface GraphQLResponse<T> {
  data?: T;
  errors?: Array<{ message: string }>;
}
```
