## Overview

The Dashboard provides a comprehensive interface for configuring alert types in FleetGuard360. Operators can create, update, and delete alert type definitions with associated priority levels and responsible areas.

## Alert Type Configuration

### Data Structure

Each alert type contains the following fields:

<ParamField path="nombre" type="string" required>
  The name of the alert type (e.g., "Exceso de velocidad", "Frenado brusco")
</ParamField>

<ParamField path="descripcion" type="string" required>
  A brief description explaining when this alert is triggered
</ParamField>

<ParamField path="nivelPrioridadId" type="number" required>
  Priority level ID:
  - `1` = Baja (Low)
  - `2` = Media (Medium)
  - `3` = Alta (High)
</ParamField>

<ParamField path="tipoEncargado" type="string" required>
  Responsible area enum value:
  - `MECANICO` - Mechanical team
  - `SEGURIDAD` - Security team
  - `OPERADOR_LOGISTICA` - Logistics operator
  - `SOPORTE_TECNICO` - Technical support
</ParamField>

## Creating Alert Types

<Steps>
  <Step title="Open Dialog">
    Click the "ADD ITEM" button in the top-right corner of the dashboard.
  </Step>
  
  <Step title="Fill Form Fields">
    Enter the alert name, description, select responsible area and priority level.
  </Step>
  
  <Step title="Save Alert">
    Click "Agregar Alerta" to create the new alert type via GraphQL mutation.
  </Step>
  
  <Step title="Confirmation">
    A success toast notification appears and the table updates with the new entry.
  </Step>
</Steps>

### Create Mutation

The system uses the following GraphQL mutation to create alert types:

```graphql Dashboard.tsx:177-199
mutation CreateTipoAlerta($input: TipoAlertaCreateInput!) {
  createTipoAlerta(input: $input) {
    id
    nombre
    descripcion
    nivelPrioridad {
      id
      nombre
    }
    tipoEncargado
  }
}
```

**Variables:**
```json
{
  "input": {
    "nombre": "Exceso de velocidad",
    "descripcion": "Vehículo excede el límite de velocidad permitido",
    "nivelPrioridadId": 3,
    "tipoEncargado": "SEGURIDAD"
  }
}
```

### Priority Color Coding

The dashboard uses color-coded badges to visually distinguish priority levels:

```tsx Dashboard.tsx:292-299
const getPriorityColor = (priorityId: number) => {
  switch (priorityId) {
    case 3: return "bg-priority-critical text-white";
    case 2: return "bg-priority-medium text-white";
    case 1: return "bg-aqua-green text-white";
    default: return "bg-muted text-muted-foreground";
  }
};
```

## Form Validation

The system validates all fields before submission:

```tsx Dashboard.tsx:138-146
if (!newAlert.name || !newAlert.description || !newAlert.priority || !newAlert.area) {
  toast({
    title: "Error",
    description: "Por favor completa todos los campos",
    variant: "destructive",
  });
  return;
}
```

<Note>
  All four fields (name, description, priority, and area) are required. The form will not submit until all fields contain valid values.
</Note>
