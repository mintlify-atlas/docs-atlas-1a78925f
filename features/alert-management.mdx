## Overview

Alert Management in FleetGuard360 enables administrators to configure how different types of alerts are categorized, prioritized, and routed to responsible teams. The system provides a complete CRUD interface for managing alert type definitions.

## Alert Type Form

The alert management form (`src/pages/Dashboard.tsx:332-416`) provides an intuitive interface for creating and editing alert configurations:

## Deleting Alert Types

<Warning>
  **Permanent Action**: Deleting an alert type cannot be undone. Ensure the alert type is no longer in use before deletion.
</Warning>

### Delete Operation

```tsx Dashboard.tsx:261-292
const handleDeleteAlert = async (id: string) => {
  try {
    const mutation = `
      mutation DeleteTipoAlerta($id: Int!) {
        deleteTipoAlerta(id: $id)
      }
    `;

    const result = await fetchGraphQL<{ deleteTipoAlerta: boolean }>(
      "/alerts/graphql",
      mutation,
      { id: parseInt(id) }
    );

    if (result.deleteTipoAlerta) {
      setAlerts(alerts.filter((alert) => alert.id !== id));
      toast({
        title: "Alerta eliminada",
        description: "La alerta ha sido eliminada del sistema",
      });
    } else {
      throw new Error("No se pudo eliminar la alerta");
    }
  } catch (error) {
    const message = error instanceof Error ? error.message : "Error al eliminar la alerta";
    toast({
      title: "Error",
      description: message,
      variant: "destructive",
    });
  }
};
```

## Priority ID Mapping

The system converts priority selections to numeric IDs:

```tsx Dashboard.tsx:125-136
const getPriorityId = (priority: string): number => {
  // Si la prioridad es un número (ID), devolverlo
  const id = parseInt(priority);
  if (!isNaN(id)) return id;
  
  // Fallback para compatibilidad
  const p = priority.toLowerCase();
  if (p.includes("alta") || p === "high") return 3;
  if (p.includes("media") || p === "medium") return 2;
  if (p.includes("baja") || p === "low") return 1;
  return 1;
};
```

All alert management operations use the `/alerts/graphql` endpoint:


The `useGraphQL` hook automatically includes JWT authentication headers for all requests (see [Authentication](/features/authentication)).
