### Export Implementation

The CSV export uses a GraphQL query to obtain a pre-signed download URL:

```tsx Estadisticas.tsx:13-61
const handleExportCSV = async () => {
  setIsExporting(true);
  try {
    toast({
      title: "Preparando exportación",
      description: "Generando archivo CSV...",
    });

    // Obtener la URL de descarga
    const query = `
      query {
        exportAlertsCsvUrl
      }
    `;

    const result = await fetchGraphQL<{ exportAlertsCsvUrl: string }>(
      "/panel/graphql",
      query
    );

    const downloadUrl = result.exportAlertsCsvUrl;

    console.log("URL de descarga:", downloadUrl);

    // Opción 1: Descargar directamente usando un link (más simple)
    const link = document.createElement("a");
    link.href = downloadUrl;
    link.download = `alertas_${new Date().toISOString().split('T')[0]}.csv`;
    link.target = "_blank";
    document.body.appendChild(link);
    link.click();
    document.body.removeChild(link);

    toast({
      title: "Exportación exitosa",
      description: "El archivo CSV ha sido descargado correctamente",
    });
    
  } catch (error) {
    console.error("Error exportando CSV:", error);
    toast({
      title: "Error",
      description: "No se pudo exportar el archivo CSV",
      variant: "destructive",
    });
  } finally {
    setIsExporting(false);
  }
};
```

## Export Process Flow

<Steps>
  <Step title="User Initiates Export">
    User clicks the "Exportar a CSV" button in the dashboard header
  </Step>
  
  <Step title="Request CSV URL">
    System sends GraphQL query to `/panel/graphql` requesting `exportAlertsCsvUrl`
  </Step>
  
  <Step title="Receive Download URL">
    Backend generates CSV file and returns a temporary download URL
  </Step>
  
  <Step title="Trigger Download">
    Client creates a temporary anchor element and programmatically clicks it to download the file
  </Step>
  
  <Step title="Success Notification">
    Toast notification confirms successful download
  </Step>
</Steps>

## GraphQL Export Query

```graphql Estadisticas.tsx:22-26
query {
  exportAlertsCsvUrl
}
```

**Response:**
```json
{
  "data": {
    "exportAlertsCsvUrl": "https://storage.example.com/exports/alertas_2026-03-04.csv?token=..."
  }
}
```

<Note>
  The backend generates a pre-signed URL for secure file access. The URL typically includes authentication tokens and may have a time-based expiration.
</Note>

## Authentication Requirements

Both the Power BI dashboard and CSV export require authenticated sessions:

```tsx Estadisticas.tsx:10
const { fetchGraphQL } = useGraphQL();
```

The `useGraphQL` hook automatically includes JWT tokens in all requests (see [Authentication](/features/authentication)).

<Note>
  If the user's session expires during export, they will be redirected to the login page with a session expiration message.
</Note>
