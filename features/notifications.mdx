## Overview

FleetGuard360 implements a real-time notification system using WebSocket connections via STOMP protocol over SockJS. The NotificationBell component (`src/components/NotificationBell.tsx`) provides instant alerts when new events occur in the fleet.

## WebSocket Hook

The `useWebSocketNotifications` hook (`src/hooks/use-websocket-notifications.ts`) manages the WebSocket connection lifecycle:

### Connection Initialization

```typescript use-websocket-notifications.ts:32-85
const connect = () => {
  try {
    // Usar ruta relativa para que funcione con el proxy de Vercel
    const socket = new SockJS("/notifications/ws-alerts");
    const stompClient = Stomp.over(socket);

    // Desactivar logs de debug (opcional)
    stompClient.debug = () => {};

    stompClient.connect(
      {},
      (frame: any) => {
        console.log("WebSocket conectado:", frame);
        setIsConnected(true);

        // Suscribirse al canal de alertas
        stompClient.subscribe("/all/alerts", (message) => {
          try {
            const alert: AlertNotification = JSON.parse(message.body);
            setNotifications((prev) => [alert, ...prev]);

            if (onNotification) {
              onNotification(alert);
            }
          } catch (error) {
            console.error("Error procesando notificación:", error);
          }
        });
      },
      (error: unknown) => {
        console.error("Error en WebSocket:", error);
        setIsConnected(false);

        // Intentar reconectar después de 5 segundos
        setTimeout(() => {
          console.log("Intentando reconectar...");
          connect();
        }, 5000);
      }
    );

    stompClientRef.current = stompClient;
  } catch (error) {
    console.error("Error al conectar WebSocket:", error);
    setIsConnected(false);
  }
};
```

<Note>
  The hook uses SockJS for WebSocket fallback compatibility and STOMP for message protocol. If the connection fails, it automatically attempts to reconnect after 5 seconds.
</Note>

```tsx NotificationBell.tsx:19-27
const { notifications, isConnected, clearNotifications } = useWebSocketNotifications({
  onNotification: (notification) => {
    // Mostrar toast cuando llegue una notificación
    toast({
      title: "Nueva Alerta",
      description: `${notification.alertType} - ${notification.generatingUnit}`,
      variant: notification.priority?.toLowerCase().includes("alta") ? "destructive" : "default",
```

## Relative Time Formatting

Notification timestamps are displayed as relative time:

```tsx NotificationBell.tsx:31-47
const formatDate = (isoDate: string) => {
  try {
    const date = new Date(isoDate);
    const now = new Date();
    const diff = now.getTime() - date.getTime();
    const minutes = Math.floor(diff / 60000);
    const hours = Math.floor(diff / 3600000);
    const days = Math.floor(diff / 86400000);

    if (minutes < 1) return "Ahora";
    if (minutes < 60) return `Hace ${minutes}m`;
    if (hours < 24) return `Hace ${hours}h`;
    return `Hace ${days}d`;
  } catch {
    return "";
  }
};
```

**Examples:**
- Just arrived: "Ahora"
- 15 minutes ago: "Hace 15m"
- 3 hours ago: "Hace 3h"
- 2 days ago: "Hace 2d"

## Priority Color Coding

Notification priorities are color-coded for quick identification:

```tsx NotificationBell.tsx:49-55
const getPriorityColor = (priority: string) => {
  const prioridadLower = priority?.toLowerCase() || "";
  if (prioridadLower.includes("alta")) return "bg-red-600 text-white";
  if (prioridadLower.includes("media")) return "bg-yellow-500 text-gray-900";
  if (prioridadLower.includes("baja")) return "bg-green-600 text-white";
  return "bg-gray-500 text-white";
};
```

The hook manages the WebSocket lifecycle with React hooks:

```typescript use-websocket-notifications.ts:100-109
useEffect(() => {
  if (autoConnect) {
    connect();
  }

  return () => {
    disconnect();
  };
  // eslint-disable-next-line react-hooks/exhaustive-deps
```

The connection is established when the component mounts and automatically cleaned up when unmounting.

## WebSocket Endpoint

Notifications are received through the WebSocket endpoint:

- **Endpoint**: `/notifications/ws-alerts`
- **Protocol**: STOMP over SockJS
- **Channel**: `/all/alerts`

```typescript
const socket = new SockJS("/notifications/ws-alerts");
const stompClient = Stomp.over(socket);
stompClient.subscribe("/all/alerts", (message) => { /* ... */ });
```
