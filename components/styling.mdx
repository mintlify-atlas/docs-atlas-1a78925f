## Overview

FleetGuard360 uses **Tailwind CSS** with custom configuration for consistent styling across the application. The design system includes custom color palettes, priority indicators, and responsive design utilities.

### Priority Colors

Special colors for alert priority levels:

```tsx
const priorityColors = {
  "priority-critical": "hsl(var(--priority-critical))", // Red for critical alerts
  "priority-high": "hsl(var(--priority-high))",         // Orange/Yellow for high priority
  "priority-medium": "hsl(var(--priority-medium))",     // Yellow for medium priority
};
```

**Usage Example:**

```tsx src/pages/Dashboard.tsx
const getPriorityColor = (priorityId: number) => {
  switch (priorityId) {
    case 3: return "bg-priority-critical text-white";
    case 2: return "bg-priority-medium text-white";
    case 1: return "bg-aqua-green text-white";
    default: return "bg-muted text-muted-foreground";
  }
};

<Badge className={getPriorityColor(alert.priorityId)}>
  {alert.priority}
</Badge>
```

## CSS Custom Properties

Colors are defined as CSS custom properties in `src/index.css`:

```css
:root {
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  
  --primary: 222.2 47.4% 11.2%;
  --primary-foreground: 210 40% 98%;
  
  --dark-blue: 207 70% 15%;        /* #0A2846 */
  --aqua-green: 174 60% 51%;
  --purple-brand: 270 50% 40%;
  --light-aqua: 180 50% 90%;
  
  --priority-critical: 0 72% 51%;   /* Red */
  --priority-high: 25 95% 53%;      /* Orange */
  --priority-medium: 48 96% 53%;    /* Yellow */
  
  --radius: 0.5rem;
}
```

<Note>
  All HSL values are defined without `hsl()` wrapper to allow alpha channel modifications: `hsl(var(--color) / 0.5)` for 50% opacity.
</Note>

## cn() Utility

The `cn()` utility function merges Tailwind classes:

```tsx lib/utils.ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

**Usage:**

```tsx
import { cn } from "@/lib/utils";

const Button = ({ className, variant, size }) => {
  return (
    <button
      className={cn(
        buttonVariants({ variant, size }),
        className
      )}
    />
  );
};
```

## Class Variance Authority

CVA is used for component variants:

```tsx
import { cva, type VariantProps } from "class-variance-authority";

const buttonVariants = cva(
  "inline-flex items-center justify-center gap-2", // Base styles
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground",
        destructive: "bg-destructive text-destructive-foreground",
      },
      size: {
        default: "h-10 px-4 py-2",
        sm: "h-9 px-3",
        lg: "h-11 px-8",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  },
);
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use Semantic Colors">
    Prefer semantic color names (`bg-primary`, `text-destructive`) over hardcoded values for theme consistency.
  </Accordion>
  
  <Accordion title="Leverage CSS Variables">
    Use CSS custom properties for colors that may change based on theme or user preferences.
  </Accordion>
  
  <Accordion title="Consistent Spacing">
    Use Tailwind's spacing scale (`p-4`, `gap-3`, `space-y-2`) for consistent spacing throughout the app.
  </Accordion>
  
  <Accordion title="Component Variants">
    Use CVA for component variants instead of conditional className strings.
  </Accordion>
</AccordionGroup>

<Note>
  Always use the `cn()` utility when combining dynamic class names to ensure proper class merging and avoid specificity conflicts.
</Note>
