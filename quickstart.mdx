## Quick Start

This guide will help you clone, configure, and run FleetGuard360 on your local machine. By the end, you'll have the application running and be able to log in to the dashboard.

<Steps>

### Clone the Repository

First, clone the FleetGuard360 frontend repository from GitHub:

```bash
git clone https://github.com/CodeFactory-FleetGuard360-EV04/Frontend.git
```

Navigate to the project directory:

```bash
cd Frontend
```

<Note>
  Make sure you have Git installed on your system. You can verify by running `git --version`.
</Note>

### Install Dependencies

FleetGuard360 requires Node.js and npm (or alternative package managers) to be installed. Install all project dependencies:

<CodeGroup>
```bash npm
npm install
```

```bash yarn
yarn install
```

```bash pnpm
pnpm install
```

```bash bun
bun install
```
</CodeGroup>

<Warning>
  Ensure you're using Node.js version 16 or higher. Check your version with `node --version`.
</Warning>

The installation will fetch all required dependencies including:

- React 18.3.1 and React DOM
- React Router v6.30.1 for navigation
- TanStack Query for data fetching
- shadcn/ui and Radix UI components
- SockJS and STOMP for WebSocket connections
- Recharts for data visualization
- TypeScript and Vite

### Start the Development Server

Start the Vite development server with hot module replacement:

<CodeGroup>
```bash npm
npm run dev
```

```bash yarn
yarn dev
```

```bash pnpm
pnpm dev
```

```bash bun
bun run dev
```
</CodeGroup>

The development server will start on **port 3000**:

```bash
  VITE v5.4.19  ready in 1247 ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: http://[::]:3000/
```

<Note>
  The server is configured to listen on all network interfaces (`::`) so you can access it from other devices on your network.
</Note>

### Open the Application

Open your web browser and navigate to:

```
http://localhost:3000
```

You should see the **FleetGuard360 login page** with a two-column layout:

- **Left column**: Branded welcome screen with the FleetGuard360 logo
- **Right column**: Login form for operator credentials

</Steps>

## First Login Workflow

Now that the application is running, let's walk through your first login.

<Steps>

### Enter Your Credentials

The login page accepts operator credentials:

1. **Email Address**: Enter your registered operator email
2. **Password**: Enter your password
3. **Remember Me** (optional): Check this to store your JWT token in `localStorage` instead of `sessionStorage`

<CodeGroup>
```typescript Login.tsx
const handleSubmit = async (e: React.FormEvent) => {
  const response = await fetch(`${API_URL}/auth/login`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      email: credentials.email,
      password: credentials.password,
    }),
  });

  const data = await response.json();
  
  if (data?.jwt) {
    // Store JWT token
    if (credentials.remember) {
      localStorage.setItem("jwt_token", data.jwt);
    } else {
      sessionStorage.setItem("jwt_token", data.jwt);
    }
    
    navigate("/dashboard");
  }
};
```
</CodeGroup>

<Note>
  The authentication endpoint is `/api/auth/login`, which is proxied to `http://fleetguard-api.ddns.net` in development mode.
</Note>

### Successful Authentication

When authentication succeeds:

1. A JWT token is stored in browser storage
2. A success toast notification appears: `"Login exitoso - Bienvenido {email}"`
3. You're automatically redirected to `/dashboard`

<Warning>
  If authentication fails, you'll see an error toast: `"Error de autenticación - Credenciales inválidas"`. Verify your credentials and try again.
</Warning>

### Explore the Dashboard

After login, you'll land on the **Dashboard** page at `/dashboard`. This is the alert configuration interface where you can:

- View all configured alert types in a table
- Create new alert types with the "ADD ITEM" button
- Edit existing alerts by clicking the edit icon
- Delete alerts by clicking the trash icon
- Configure priority levels (High, Medium, Low)
- Assign responsible areas (Mechanic, Security, Logistics, Technical Support)

### Real-Time Notifications

FleetGuard360 automatically connects to the WebSocket server when you log in. The connection is established using:

- **Protocol**: SockJS + STOMP
- **Endpoint**: `/notifications/ws-alerts`
- **Channel**: `/all/alerts`

Notifications appear instantly without requiring page refreshes.

<Note>
  The WebSocket connection automatically reconnects if it's interrupted, with a 5-second retry interval.
</Note>

## Common Issues

<AccordionGroup>
  <Accordion title="Port 3000 already in use">
    If port 3000 is occupied, Vite will automatically try the next available port (3001, 3002, etc.). Check the terminal output for the actual port.
    
    You can also specify a custom port in `vite.config.ts`:
    
    ```typescript
    export default defineConfig({
      server: {
        port: 3001,
      },
    });
    ```
  </Accordion>
  
  <Accordion title="API proxy connection refused">
    The development server proxies API requests to `http://fleetguard-api.ddns.net`. If you see connection errors:
    
    - Verify the backend server is running
    - Check network connectivity
    - Ensure no firewall is blocking the connection
    - Confirm the backend URL in `vite.config.ts:13`
  </Accordion>
  
  <Accordion title="WebSocket connection failed">
    WebSocket connections require the `/notifications` proxy to be working. Check:
    
    - Backend WebSocket endpoint is accessible
    - Proxy configuration in `vite.config.ts:22-27`
    - Browser console for detailed error messages
  </Accordion>
  
  <Accordion title="Module not found errors">
    If you see import errors after cloning:
    
    1. Delete `node_modules` folder
    2. Delete package lock files (`package-lock.json`, `yarn.lock`, etc.)
    3. Run `npm install` again
    4. Clear Vite cache: `rm -rf node_modules/.vite`
  </Accordion>
</AccordionGroup>
