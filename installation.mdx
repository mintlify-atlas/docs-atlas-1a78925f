## Installation Steps

Follow these steps to install FleetGuard360 on your local machine.

<Steps>

### Clone the Repository

Clone the FleetGuard360 frontend repository:

```bash
git clone https://github.com/CodeFactory-FleetGuard360-EV04/Frontend.git
cd Frontend
```

The repository structure:

```
Frontend/
├── src/
│   ├── components/     # Reusable UI components
│   ├── pages/         # Route components
│   ├── hooks/         # Custom React hooks
│   ├── lib/           # Utility functions
│   └── assets/        # Images and static files
├── public/            # Public static assets
├── package.json       # Dependencies and scripts
├── vite.config.ts     # Vite configuration
├── tsconfig.json      # TypeScript configuration
├── tailwind.config.js # Tailwind CSS configuration
└── vercel.json        # Vercel deployment config
```

### Configure Environment

FleetGuard360 uses Vite's configuration file for environment setup. The proxy configuration is already set up in `vite.config.ts`:

<CodeGroup>
```typescript vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react-swc";
import path from "path";

export default defineConfig(({ mode }) => ({
  server: {
    host: "::",
    port: 3000,
    proxy: {
      "/api": {
        target: "http://fleetguard-api.ddns.net",
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ""),
        configure: (proxy, _options) => {
          proxy.on("proxyReq", (proxyReq, _req, _res) => {
            proxyReq.setHeader("Origin", "http://fleetguard-api.ddns.net");
          });
        },
      },
      "/notifications": {
        target: "http://fleetguard-api.ddns.net",
        changeOrigin: true,
        ws: true, // Enable WebSocket proxy
        secure: false,
      },
    },
  },
  plugins: [react()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
}));
```
</CodeGroup>

**Key Configuration Points:**

- **Host**: `::` - Listens on all network interfaces (IPv6)
- **Port**: `3000` - Development server port
- **API Proxy**: `/api/*` routes to `http://fleetguard-api.ddns.net`
- **WebSocket Proxy**: `/notifications/*` routes with WebSocket support
- **Path Alias**: `@` resolves to `./src`

<Warning>
  The backend API URL is hardcoded to `http://fleetguard-api.ddns.net`. Update this if your backend is hosted elsewhere.
</Warning>

## Build Modes

FleetGuard360 supports different build modes for various deployment scenarios.

### Development Build

Create a development build with source maps and debugging:

<CodeGroup>
```bash npm
npm run build:dev
```

```bash yarn
yarn build:dev
```

```bash pnpm
pnpm build:dev
```

```bash bun
bun run build:dev
```
</CodeGroup>

This runs `vite build --mode development` and creates an unoptimized build in the `dist/` directory.

### Production Build

Create an optimized production build:

<CodeGroup>
```bash npm
npm run build
```

```bash yarn
yarn build
```

```bash pnpm
pnpm build
```

```bash bun
bun run build
```
</CodeGroup>

Production builds include:

- **Minification**: JavaScript and CSS minification
- **Tree Shaking**: Removal of unused code
- **Code Splitting**: Optimized chunk splitting
- **Asset Optimization**: Image and font optimization

<Note>
  Production builds are placed in the `dist/` directory and are ready for deployment.
</Note>

### Preview Production Build

Test the production build locally before deploying:

<CodeGroup>
```bash npm
npm run preview
```

```bash yarn
yarn preview
```

```bash pnpm
pnpm preview
```

```bash bun
bun run preview
```
</CodeGroup>

This starts a local server to preview the production build at `http://localhost:4173`.

## Vercel Deployment Configuration

FleetGuard360 includes a production-ready Vercel deployment configuration:

<CodeGroup>
```json vercel.json
{
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "http://fleetguard-api.ddns.net/:path*"
    },
    {
      "source": "/notifications/:path*",
      "destination": "http://fleetguard-api.ddns.net/notifications/:path*"
    }
  ]
}
```
</CodeGroup>

**Deployment Steps:**

<Steps>

### Install Vercel CLI

```bash
npm install -g vercel
```

### Deploy to Vercel

From the project directory:

```bash
vercel
```

Follow the prompts to:
1. Link to your Vercel account
2. Configure project settings
3. Deploy the application

### Production Deployment

Deploy to production:

```bash
vercel --prod
```

</Steps>

<Warning>
  Vercel's free tier has limitations on serverless function execution time. WebSocket connections may require an upgraded plan.
</Warning>

## Environment Variables

While FleetGuard360 doesn't use a `.env` file by default, you can add environment variables for customization:

```bash .env.local
# Backend API URL (optional - defaults to proxy)
VITE_API_URL=http://fleetguard-api.ddns.net

# WebSocket URL (optional)
VITE_WS_URL=http://fleetguard-api.ddns.net/notifications

# Application Mode
VITE_APP_MODE=production
```

<Note>
  Vite requires environment variables to be prefixed with `VITE_` to be exposed to the client.
</Note>

## Troubleshooting Installation Issues

<AccordionGroup>
  <Accordion title="npm install fails with ERESOLVE errors">
    This usually indicates dependency conflicts. Try:
    
    ```bash
    npm install --legacy-peer-deps
    ```
    
    Or use the `--force` flag (use with caution):
    
    ```bash
    npm install --force
    ```
  </Accordion>
  
  <Accordion title="TypeScript errors during build">
    Ensure you're using TypeScript 5.8.3 or compatible:
    
    ```bash
    npm install typescript@^5.8.3 --save-dev
    ```
    
    Then rebuild:
    
    ```bash
    npm run build
    ```
  </Accordion>
  
  <Accordion title="Vite dev server not starting">
    Common causes:
    
    1. **Port already in use**: Change the port in `vite.config.ts`
    2. **Permission issues**: Run with appropriate permissions
    3. **Node version**: Ensure Node.js 16+ is installed
    
    Check for errors in the terminal output.
  </Accordion>
  
  <Accordion title="Proxy requests failing (401/403/CORS)">
    If API requests fail:
    
    1. **Verify backend is running**: `curl http://fleetguard-api.ddns.net`
    2. **Check proxy config**: Review `vite.config.ts` proxy settings
    3. **CORS headers**: Ensure backend allows the origin
    4. **JWT token**: Verify token is stored correctly in browser storage
    
    Check browser DevTools Network tab for detailed error messages.
  </Accordion>
  
  <Accordion title="WebSocket connection refused">
    WebSocket issues are often related to:
    
    1. **Backend not supporting WebSocket**: Verify STOMP endpoint
    2. **Proxy misconfiguration**: Check `ws: true` in proxy config
    3. **Firewall blocking**: Ensure WebSocket traffic is allowed
    4. **Protocol mismatch**: Verify SockJS compatibility
    
    Check browser console for WebSocket error logs.
  </Accordion>
  
  <Accordion title="Build size too large">
    If your production build is larger than expected:
    
    1. **Analyze bundle**: Use `vite-plugin-visualizer`
    2. **Remove unused dependencies**: Check for unnecessary imports
    3. **Lazy load routes**: Implement code splitting
    4. **Optimize images**: Compress assets before importing
    
    Run the build with analysis:
    
    ```bash
    npm run build -- --mode analyze
    ```
  </Accordion>
</AccordionGroup>
