## What is FleetGuard360?

FleetGuard360 is an **Intelligent Satellite Monitoring System** designed specifically for passenger transport operations. It provides real-time fleet tracking, alert management, and comprehensive analytics to help transport operators maintain safety, efficiency, and operational excellence.

<Note>
  FleetGuard360 combines real-time data processing, WebSocket notifications, and GraphQL APIs to deliver instant insights into your fleet operations.
</Note>

## Architecture Overview

FleetGuard360 is built on modern web technologies to ensure performance, scalability, and developer experience:

- **Frontend Framework**: React 18.3 with TypeScript for type-safe development
- **Build Tool**: Vite 5.4 for lightning-fast builds and hot module replacement
- **UI Components**: shadcn/ui + Radix UI for accessible, customizable components
- **Styling**: Tailwind CSS for utility-first styling
- **State Management**: TanStack Query (React Query) for server state
- **Real-Time**: SockJS + STOMP for WebSocket connections
- **API Layer**: GraphQL for efficient data fetching
- **Routing**: React Router v6 for client-side navigation

## Core Capabilities

### Alert Management System

FleetGuard360's alert system supports:

- **Configurable Alert Types**: Create custom alert types with names, descriptions, and responsible areas
- **Priority Levels**: Three-tier priority system (High, Medium, Low) with visual indicators
- **Responsible Areas**: Route alerts to specific teams (Mechanic, Security, Logistics, Technical Support)
- **Alert States**: Track alert lifecycle (Pending, In Progress, Resolved)
- **Historical Records**: Complete audit trail of all alerts with timestamps

<Note>
  Alert configurations are managed through the Dashboard page (`/dashboard`) and use GraphQL mutations for CRUD operations. See [Alert Management](/features/alert-management) for details.
</Note>

### Real-Time Notification Engine

The notification system leverages WebSocket technology:

```typescript src/hooks/use-websocket-notifications.ts
import SockJS from "sockjs-client";
import Stomp from "stompjs";

const socket = new SockJS("/notifications/ws-alerts");
const stompClient = Stomp.over(socket);

stompClient.subscribe("/all/alerts", (message) => {
  const alert = JSON.parse(message.body);
  // Handle real-time alert notification
});
```

See [WebSocket Integration](/integration/websocket) for implementation details.

### GraphQL API Integration

All data operations use GraphQL for efficient querying:

```typescript src/hooks/use-graphql.ts
const { fetchGraphQL } = useGraphQL();

const query = `
  query GetAllAlerts {
    getAllAlerts {
      id
      alertType
      priority
      state
      generationDate
    }
  }
`;

const result = await fetchGraphQL("/panel/graphql", query);
```

Explore the [GraphQL API Reference](/api/overview) for all available queries and mutations.

## Application Routes

FleetGuard360 includes the following main routes:

| Route | Component | Description |
|-------|-----------|-------------|
| `/` | Login | Authentication page for operators |
| `/dashboard` | Dashboard | Alert configuration and management |
| `/panel-de-alertas` | PanelDeAlertas | Alert history and filtering |
| `/estadisticas` | Estadisticas | Statistics and analytics |

<Warning>
  All routes except `/` require JWT authentication. Unauthenticated users are redirected to the login page.
</Warning>
